Per-Node Assumptions

  • ~150 critical tables monitored per node (freshness, volume, schema via ML)
  • ~500 metrics per node (metrics anomalies, distribution changes, comparison checks...) - important: metrics broken down by a dimension (e.g. market, product) count as individual metrics.
  • ~100 custom SQL / validation rules per node (complex checks)
Assumption is that nodes and markets follow a long tail pattern, with some nodes having a leading adoption and others having a minimal use.
Additional credits can be purchased at any time during the term at the same discounted rate, in 100,000-credit increments.

Rate source: Monte Carlo Consumption Rates - Version 2.1.
